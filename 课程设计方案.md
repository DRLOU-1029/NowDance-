# NowDance舞蹈动作评分系统课程设计方案

## 一、项目背景与目标

### 1.1 项目背景
随着体感游戏的普及,以《舞力全开》(Just Dance)为代表的舞蹈游戏受到广泛欢迎。本课程设计旨在开发一个基于计算机视觉的舞蹈动作评分系统,通过实时捕捉玩家姿态并与标准动作进行比对,实现客观、量化的舞蹈评分。

### 1.2 设计目标
- 实现从视频中提取人体姿态关键点
- 设计姿态归一化算法,消除距离、角度等环境因素影响
- 采用DTW算法解决玩家动作速度不一致问题
- 实现多维度评分机制(关键点位置+关节角度)
- 支持实时摄像头评分功能

## 三、核心算法设计

### 3.1 姿态关键点提取
采用MediaPipe Pose提取33个人体关键点,包括:
- 面部关键点(鼻、眼、耳、嘴)共10个
- 上肢关键点(肩、肘、腕、指)共12个
- 下肢关键点(髋、膝、踝、足)共11个

每个关键点包含:
- (x, y): 归一化图像坐标 [0,1]
- z: 深度近似值
- visibility: 可见度置信度

### 3.2 姿态归一化算法
为消除摄像头距离、人物站位等环境因素影响,设计三层归一化:

**步骤1: 原点对齐**
```
原点 = (左髋 + 右髋) / 2
所有关键点 -= 原点
```

**步骤2: 尺度归一化**
```
身体尺度 = mean(肩宽, 髋宽, 左躯干长, 右躯干长)
所有关键点 /= 身体尺度
```

**步骤3: 可见度加权**
```
归一化坐标 *= visibility
```

### 3.3 帧级相似度计算

**关键点距离分数:**
```
距离分数 = 1 - mean(||p_i - q_i||) / 最大距离阈值
```

**关节角度分数:**
```
角度分数 = mean(cos(θ_i - θ'_i))
```
主要关节包括:肩、肘、髋、膝

**综合评分:**
```
总分 = 0.7 × 距离分数 + 0.3 × 角度分数
```

### 3.4 DTW时间对齐
采用动态时间规整算法解决玩家动作速度不一致问题:

**代价矩阵:**
```
D(i,j) = d(frame_i, frame_j) + min(D(i-1,j), D(i,j-1), D(i-1,j-1))
```

**对齐路径:**
通过回溯找到最小代价路径,实现动作序列的弹性匹配。

## 四、评级规则设计

根据相似度分数映射为游戏评级:

| 分数区间 | 评级 | 说明 |
|---------|------|------|
| 90-100 | Perfect | 动作完全同步 |
| 75-89 | Great | 动作高度相似 |
| 60-74 | Good | 动作基本正确 |
| 40-59 | Okay | 动作部分错误 |
| 0-39 | Miss | 动作严重错误 |

## 五、实时评分机制

### 5.1 节拍判定系统
- BPM参数:每分钟节拍数
- 判定窗口:节拍前后时间容忍度(默认±0.35秒)
- 评分策略:每N拍结算一次

### 5.2 连击系统
```
if 当前节拍分数 >= hit_threshold:
    combo += 1
else:
    combo = 0
```

### 5.3 性能优化
- 推理分辨率降采样(inference-width=360)
- 帧跳过策略(process-every=2)
- 上半身轻量模式(仅使用肩肘腕髋)

## 八、创新点与难点

### 8.1 创新点
1. **多维度评分**:结合关键点位置和关节角度,评分更科学
2. **弹性时间对齐**:DTW算法容忍玩家速度差异
3. **节拍判定机制**:类似音游的节拍评分系统
4. **实时可视化**:骨架叠加、分数实时显示

### 8.2 技术难点
1. 姿态遮挡问题(侧面、背面)
2. 快速动作的帧率需求
3. 实时性能优化
4. 不同体型的适应性
